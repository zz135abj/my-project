# 新手学习指南

欢迎来到 Python 与 PyTorch 入门教程！本指南面向第一次接触本项目的初学者，帮助你规划学习节奏、理解各类文档，以及高效利用练习与项目。

## 指南目标

- 清楚了解仓库结构和每份文档的作用
- 根据自己的时间安排制定合理的学习计划
- 掌握每节课的推荐学习流程，避免“只看不练”
- 借助配套工具跟踪进度、记录反思，稳步提升

## 步骤 3：每节课的推荐学习流程

按照以下清单推进，能有效避免“看懂了但不会写”：

- [ ] **预习**：浏览 `COURSE_OUTLINE.md` 或该课的 README，了解本节目标与练习
- [ ] **理解**：逐段阅读课程 README，结合笔记记录关键概念与示例
- [ ] **运行示例**：亲自执行 `demo.py` 或 `pytorch_demo.py`，观察输出与代码结构
- [ ] **拆解练习**：阅读 `exercises.py` 中的 TODO 注释，明确需要完成的任务
- [ ] **独立实现**：尝试自己完成练习；如遇困难，可返回课程内容或搜索资料
- [ ] **查漏补缺**：对照示例代码或参考答案（若有），检查自己的实现是否合理
- [ ] **复盘记录**：在笔记或 `PROGRESS_TRACKER.md` 中写下收获、疑问和下次改进点

---

## 步骤 4：充分利用练习与笔记

1. **练习优先**：练习题设计层层递进，优先确保练习完成率高于“阅读次数”。
2. **番茄工作法**：25 分钟专注编写练习，5 分钟整理笔记或放松，有助于保持节奏。
3. **结构化笔记**：建议按照“概念解释 → 示例代码 → 常见错误”三栏记录，便于回顾。
4. **定期复习**：每完成一个模块（例如 lesson01-07），抽 30 分钟回顾要点、整理错题。

## 遇到困难怎么办？

- **先复现问题**：把报错信息、输入数据、环境信息记录下来，方便搜索或求助
- **复查文档**：回看课程 README、官方文档或 `SETUP.md` 中的说明，确认操作是否正确
- **缩小问题范围**：用最小可复现代码（Minimal Reproducible Example）验证自己的推测
- **使用社区资源**：善用官方论坛、Stack Overflow、GitHub Issue、B 站课程等渠道
- **给自己冷静期**：暂时离开 10-15 分钟，再回来往往更容易找到疏漏
